# 设置面板供应商面板为空 — 问题修复与分析

## 一、问题总结

用户打开设置菜单 → 供应商面板时，面板显示"暂无供应商" / "请选择一个供应商"的空白状态。排查后确定是两个问题叠加导致：数据层的静默丢失和 UI 层的自毁式隐藏。

## 二、问题分析

### 问题 1：`loadSettings()` 后端有空数据时跳过 localStorage 降级

**文件**: `src/store.js:387-401`

**现象**: 后端 DB 中 settings 的 `providers` 字段为 `{}` 时，前端只拿到空数据，localStorage 中的备份被忽略。

**原因**: 原代码结构：

```js
async function loadSettings() {
  try {
    return await apiGet('/api/settings') || {};  // 后端返回 {} → 直接 return
  } catch (e) { ... }
  // ↓ 只有 fetch 抛异常才会走到这里
  try { ... localStorage ... }
  return {};
}
```

`{}` 在 JS 中是 truthy，`{} || {}` 短路求值为第一个 `{}`，函数直接 return，不会执行下面的 localStorage 降级。只有当 fetch 本身抛异常（网络不通、后端挂了）时，catch 块才会触发，才会走 localStorage。

**后果**: 后端运行正常但 DB 中 settings 为空 → 前端拿到 `providers: {}` → `state.providers` 保持 `{}` → 任何后续 `saveSettings()` 都会把空的 `providers: {}` 写回后端，**覆盖**掉后端可能残留的数据，造成永久丢失。

### 问题 2：`showProviderPanel()` 空态时隐藏了添加按钮

**文件**: `src/components/settingsModal.js:64-87`

**现象**: providers 为空时，用户看不到任何可操作的元素。

**原因**: HTML 结构中 "+" 添加按钮在左侧面板 `#settingsProviderPanel` 内部：

```html
<div id="settingsProviderPanel">            <!-- 这整块被 display:none -->
  <button id="settingsSidebarAdd">+</button> <!-- 添加按钮也被隐藏 -->
  <div id="settingsProviderItems"></div>
</div>
```

原代码在 `names.length === 0` 时执行 `panel.style.display = 'none'`，把整块面板连同按钮一起隐藏。

**后果**: 用户看到"请选择一个供应商"的提示，但左侧列表为空且不显示按钮，无法自行添加供应商，形成死胡同。

### 问题 3：`debouncedBackendSync()` 静默 catch

**文件**: `src/store.js:409`

```js
try { await apiPost('/api/sessions', ...); } catch { }
```

sessions POST 失败时没有任何日志，无法排查后端同步问题。settings POST 的 catch 有日志但 sessions 没有，不一致。

## 三、解决方案

### 修复 1：`loadSettings()` 始终合并 localStorage

```js
async function loadSettings() {
  let result = {};
  try {
    result = await apiGet('/api/settings') || {};
  } catch (e) { ... }
  // 始终尝试从 localStorage 合并
  try {
    const localData = localStorage.getItem('image-gen-settings-v2');
    if (localData) {
      const parsed = JSON.parse(localData);
      result = { ...parsed, ...result };  // localStorage 先展开，后端覆盖
    }
  } catch (e) { ... }
  return result;
}
```

`{ ...parsed, ...result }` 的含义：localStorage 的数据先展开，后端数据后展开覆盖同名字段。因此：
- 后端有 providers → 用后端的
- 后端 providers 为空 → localStorage 的 providers 被保留（因为我们对 result 的展开在后）
- 后端完全不可用 → result 为 `{}`，全部用 localStorage

**注意**: 这是一个浅合并。`providers` 作为一个整体对象被覆盖，不会深度合并。例如后端有 `BackendProvider`、localStorage 有 `OldProvider`，最终只保留 `BackendProvider`。这是正确的行为——后端为权威数据源。

### 修复 2：`showProviderPanel()` 空态不隐藏面板

```js
if (names.length === 0) {
  empty.style.display = 'flex';      // 右侧显示引导提示
  panel.style.display = 'flex';      // 左侧面板保留（"+ "按钮可见）
  renderProviderList();              // 渲染"暂无供应商"提示
} else {
  empty.style.display = 'none';
  panel.style.display = 'flex';
  renderProviderList();
}
```

关键改动：空态时 `panel.style.display = 'flex'` 而非 `'none'`。

### 修复 3：补充静默 catch 日志

```js
try { await apiPost('/api/sessions', ...); } catch (e) { console.error('[同步] 保存 sessions 到后端失败:', e); }
```

## 五、给后代 Claude 的线索

### 数据流关键路径

```
页面加载 → initStore() → loadSettings() → apiGet('/api/settings')
                                              ↓ 失败
                                         localStorage 降级
                                              ↓
                                       state.providers = ...
                                              ↓
用户点击设置 → openSettingsModal() → showProviderPanel() → 左侧面板 + 右侧内容
```

### 踩坑记录

1. **`{}` 是 truthy**：不要用 `||` 判断后端返回的空对象。正确的降级模式是始终合并 localStorage，后端覆盖。

2. **UI 元素层级关系**：`#settingsSidebarAdd` 按钮在 `#settingsProviderPanel` 内部，隐藏面板 = 隐藏按钮。改 DOM 可见性时检查子元素。

3. **浅合并行为**：`{ ...local, ...backend }` 只合并第一层。`providers` 对象整体替换，不是深度合并。这是故意的——后端为权威。

4. **数据覆盖链**：`saveSettings()` 每次都把完整的 `state.providers` 写入后端。如果 `state.providers` 为空，就会覆盖掉后端数据。必须确保 `loadSettings()` 正确恢复数据后再触发保存。

2. `[DEBUG initStore]` — settings 最终是什么？
3. `[DEBUG openSettingsModal]` — 打开面板时 providers 是否为空？
